docker version

sudo docker version

--> gives docker version information.


sudo docker run -dit debian

--> Runs the docker command for debain image.

docker ps
--> Gives a list of docker containers.

sudo docker images
--> Gives a list of docker images.

docker stop cf0638282adb
--> Stops the docker container based on the id.

sudo docker inspect 0d01188e8dd0
--> Used for inspecting docker container using specific hash.

docker pull nginx
--> Pulls the nginx image from dockerhub.

sudo docker history nginx
--> Gives the history of the docker image in question.

sudo docker images --no-trunc
--> No truncation of information.

docker tag --help
--> Gives information about the tag command of docker.

docker tag nginx:latest nginx:myblog_stable
--> Creates an alias using the source tag name 'latest' to target tag name 'myblog_stable'.

more Dockerfile
--> Used to see the contents of dockerfile. 'more' command is specific to windows. For Linux, please use 'less'.

 docker build -t mynginx .
 --> Used for building an image from a Dockerfile locally present in the folder from which the command is executed via the command prompt.
 -t allows to name a tag for our image.
 . tells the docker to use the Dockerfile in the current directory.

docker rmi nginx:myblog_stable
--> Used for removing an image with image name 'nginx:myblog_stable'.

docker image prune
--> Cleans up disk space.

 docker system df
 --> Check the disk free on the system

 docker search
 --> Search Docker Hub for images

sudo docker run -dit --name=web debian
--> Using --name=web, we can name the docker container as web.

sudo docker ps -a
--> Shows all containers(running as well as stopped) in the system.

docker stop web
--> Stops the container with the name web.

docker rm web
--> Removes a container of name web.

sudo docker run -dit --restart=always --name=umer2 debian
--> Using this command, docker will restart the container incase of a server failure and subsequent bootup
without the need for manual human intervention.

sudo docker inspect umer2 | grep -A3 RestartPolicy
--> sudo docker inspect umer2: Used for inspecting the container named 'umer2'
--> grep -A3 RestartPolicy: grabs the word 'RestartPolicy'.
    -A flag not only grabs the matching line but also 3 lines after it.

docker logs -tf 4f353
--> Logs the container with id given.t=Timestamp f=follow(similar to tail -f)
